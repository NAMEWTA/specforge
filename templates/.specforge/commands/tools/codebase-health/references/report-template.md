# 健康报告输出模板

> 本文档定义 `codebase-health` 产出的 Markdown 报告结构。

## 报告路径

`specforge/context/health/<YYYY-MM-DD>-report.md`

## 模板结构

```markdown
# 健康巡检报告 · YYYY-MM-DD

## 综合评分

- 当前：XX/100
- 上次：YY/100（YYYY-MM-DD）（首次巡检标注「无历史基线」）

## 探针执行摘要

| 探针 | 状态 | 评分 | 发现数 |
|------|------|------|--------|
| jscpd | ✅ 可用 / ❌ 不可用 | XX/100 | N 条 |
| knip | ✅ / ❌ | XX/100 | N 条 |
| vulture | ✅ / ❌ | XX/100 | N 条 |
| staticcheck | ✅ / ❌ | XX/100 | N 条 |
| baseline | ✅（兜底） | XX/100 | N 条 |

## 高优先级问题（High）

- `<file>:<line-range>` — <描述>
- ...

## 中优先级问题（Medium）

- `<file>:<line-range>` — <描述>
- ...

## 低优先级问题（Low）

- `<file>` — <描述>
- ...

## 未用依赖

- <package-name>（处理状态：已写入禁动清单 / 仅记录 / 已忽略）
- ...

## 改善建议

1. <建议内容>
2. ...

## 元数据

- 扫描时间：YYYY-MM-DDTHH:mm:ssZ
- 根目录：<rootDir>
- 运行模式：interactive / non-interactive / json
```

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| 综合评分 | 是 | 所有可用探针 score 加权平均 |
| 探针执行摘要 | 是 | 每个探针的状态与评分 |
| 高/中/低优先级问题 | 是 | 按严重度分组的 topIssues（允许为空） |
| 未用依赖 | 条件 | 仅 knip 可用且发现未用依赖时出现 |
| 改善建议 | 是 | 按 score 从低到高排列 |
| 元数据 | 是 | 扫描上下文信息 |

## 与历史报告对比

第二次及以后的巡检，报告顶部「综合评分」段须包含与上次的对比（↑ 改善 / ↓ 退化 / → 持平）。首次巡检标注「无历史基线」。
